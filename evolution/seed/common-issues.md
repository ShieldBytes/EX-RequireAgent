# 高频遗漏清单（种子数据）

> 出厂内置的常见需求遗漏，帮助完整性 Agent 在新项目中快速发现问题。
> 按出现频率排序，排在前面的是几乎所有项目都会遗漏的。

## 几乎所有项目都会遗漏

- **错误处理和异常恢复**：网络中断、服务超时、数据异常时的用户提示和恢复机制
- **空状态展示**：列表无数据时、搜索无结果时、首次使用无内容时的页面展示
- **首次使用引导**：新用户的欢迎流程、功能引导、快速上手教程
- **操作反馈**：每个用户操作后是否有成功/失败提示，加载状态展示
- **数据备份和恢复**：用户数据如何备份、丢失后如何恢复
- **离线/弱网模式**：无网络时哪些功能可用、数据如何同步
- **批量操作**：是否需要批量删除、批量编辑、批量导出等
- **并发冲突**：多人或多设备同时操作同一数据时如何处理
- **操作日志**：关键操作是否有记录，用于审计和问题排查

## 涉及用户系统时常遗漏

- **账号注销**：注销流程、数据处理规则、冷静期
- **密码找回**：通过手机/邮箱找回、安全验证流程
- **多设备登录**：是否允许、如何提示、如何强制下线
- **账号冻结/解冻**：触发条件、冻结期间的权限、解冻流程
- **个人信息修改**：哪些信息可改、修改是否需要验证、修改历史
- **用户反馈渠道**：意见反馈、问题举报、客服入口

## 涉及支付时常遗漏

- **支付失败处理**：重试机制、超时处理、用户提示
- **退款流程**：退款条件、审核流程、退款时限、退回渠道
- **支付对账**：交易记录核对、异常交易处理
- **多支付方式**：不同支付方式的差异处理
- **支付安全**：防重复支付、防篡改、风控规则

## 涉及内容/数据管理时常遗漏

- **数据导入导出**：支持的格式、大数据量处理、导出限制
- **数据删除策略**：物理删除 vs 逻辑删除、数据保留期限
- **版本历史**：内容的修改历史、版本回滚
- **搜索和筛选**：搜索范围、筛选条件、排序规则
- **数据分页**：大列表的分页策略、加载策略

## 涉及通知/消息时常遗漏

- **通知渠道**：站内信、推送、短信、邮件的选择和优先级
- **通知频控**：防止频繁打扰的限制规则
- **通知偏好设置**：用户可自定义接收哪些通知
- **消息已读/未读**：状态管理和展示

## 涉及权限时常遗漏

- **权限继承**：组织架构中的权限传递
- **权限变更**：权限调整后已登录用户的处理
- **临时权限**：限时授权、权限到期处理
- **越权防护**：防止通过 URL 或接口越权访问
