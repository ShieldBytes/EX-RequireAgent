# 需求追踪

你需要追踪某个需求或功能从首次出现到最终版本的完整变更历史。

## 执行步骤

### 1. 解析参数

从用户输入 `$ARGUMENTS` 中解析搜索关键词。

如果参数缺失，提示用户：
```
用法：/require-trace <关键词>
示例：/require-trace 退款功能
示例：/require-trace 用户认证
```

### 2. 定位项目

扫描 `.require-agent/projects/` 目录，找到最近修改的项目。
读取 state.json 获取项目信息。

### 3. 搜索版本文件

使用 Glob 获取 `versions/` 目录下所有版本文件列表。

对每个版本文件，使用 Grep 搜索关键词：
- 记录关键词首次出现的版本
- 记录每个版本中包含关键词的行和上下文
- 对比相邻版本中关键词相关内容的变化

### 4. 搜索评分记录

在 `scores/` 目录中搜索与关键词相关的评分变更：
- 检查各轮次评分中是否有涉及该关键词的维度变化
- 记录与该需求相关的分数变动

### 5. 搜索轮次历史

读取 state.json 中的 `round_history`，查找：
- 哪些轮次涉及了该关键词相关的优化
- 各轮次的 action 和 score_change

### 6. 构建时间线

将所有搜索结果按时间顺序组织为完整时间线。

### 7. 展示结果

```
🔍 需求追踪：「{关键词}」

项目：{项目名}
首次出现：v{N}（第 {N} 轮）
最新版本：v{M}

📅 变更时间线：

v{N} — 第 {N} 轮 — {时间}
  首次引入
  内容：{关键词相关内容摘要}

v{N+1} — 第 {N+1} 轮 — {时间}
  {变更类型：新增/修改/扩展/重构}
  变更：{具体变化描述}
  评分影响：{相关维度} {+/-分数}

{...更多版本}

v{M} — 第 {M} 轮 — {时间}（当前）
  最终状态
  内容：{当前版本中的完整内容}

📊 统计
  涉及版本数：{N}
  总变更次数：{N}
  相关维度影响：{受影响的维度列表}
```
