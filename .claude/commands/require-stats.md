# 全局优化统计

你需要汇总所有需求优化项目的数据，生成全局统计报告。

## 执行步骤

### 1. 扫描所有项目

使用 Glob 扫描以下路径，收集所有项目数据：
- `.require-agent/projects/*/state.json` — 项目状态
- `.require-agent/projects/*/scores/*.json` — 所有评分记录

如果没有找到任何项目，提示：
```
未找到任何需求优化项目，无统计数据。
使用 /require "需求描述" 启动第一个项目。
```

### 2. 读取所有项目数据

逐个读取每个项目的 `state.json` 和 `scores/` 下的所有评分文件。

### 3. 计算统计指标

#### a) 基础统计

- **总项目数**：项目目录总数
- **已完成项目数**：`phase === "completed"` 的项目数
- **进行中项目数**：其余项目数
- **完成率**：已完成 / 总项目数

#### b) 轮次统计

- **平均轮次**：所有已完成项目的 `current_round` 平均值
- **最少轮次**：已完成项目中最少的轮次数及项目名
- **最多轮次**：已完成项目中最多的轮次数及项目名

#### c) 评分统计

- **平均初始分**：所有项目 `round-0.json` 的 `average` 平均值
- **平均最终分**：所有已完成项目最终评分的 `average` 平均值
- **平均提升幅度**：最终分 - 初始分 的平均值
- **最高最终分**：所有项目中最高的最终平均分及项目名

#### d) 维度统计

对每个维度（completeness、consistency、feasibility、user_journey、business_loop 以及按需维度）：
- 计算所有项目中该维度的平均初始分和平均最终分
- 找出提升幅度最大和最小的维度

#### e) 策略效果统计（最有效策略 TOP5）

遍历所有项目的 `round_history`，统计每种 `target_dimension` + `action` 组合的出现次数和成功率（`action === "improved"` 视为成功）：

按成功率排序，取 TOP5。

#### f) 停滞统计

- **最常停滞维度**：在所有项目中出现在 `stalled_dimensions` 中次数最多的维度
- **从未停滞维度**：从未出现在任何项目的 `stalled_dimensions` 中的维度

### 4. 展示统计报告


## --team 模式：团队统计看板

如果用户输入包含 `--team` 参数，跳过上述本地统计逻辑，执行以下团队统计流程。
如果用户没有 `--team` 参数，保持上述本地统计逻辑不变。

### 1. 拉取共享数据

读取 `.require-agent/config.json` 的 `evolution_repo` 字段。

- 如果 `evolution_repo` 为空或不存在 → 输出 `"未配置共享仓库，请先在 config.json 中设置 evolution_repo。"` 并终止。
- 如果有值，使用 Bash 执行浅克隆：
  ```bash
  mkdir -p .require-agent/sync-tmp
  git clone --depth 1 {evolution_repo} .require-agent/sync-tmp/shared-data
  ```
- 读取 `.require-agent/sync-tmp/shared-data/` 下所有进化数据文件。

### 2. 成员统计

扫描共享仓库中所有项目日志文件，按 `user` 字段分组统计。

对每位成员计算：
- **项目数**：该成员参与的项目总数
- **平均满意度**：该成员所有项目最终 `satisfaction` 的平均值
- **最擅长维度**：该成员在所有维度中平均提升幅度最大的维度

输出格式：
```
👥 团队统计

成员：{N} 人 | 总项目：{M} 个

| 成员 | 项目数 | 平均满意度 | 最擅长维度 |
|------|--------|-----------|-----------|
| {user} | {count} | {avg_sat} | {best_dim} |
| ... | ... | ... | ... |
```

### 3. 团队最有效策略 TOP5

从共享仓库中所有策略记录聚合，按多因子推荐分排序（综合成功率、使用次数、平均提升分），取 TOP5。

输出格式：
```
📊 团队最有效策略 TOP5：
1. {策略名}（{维度}）— 成功率 {X}%，平均提升 {Y} 分
2. {策略名}（{维度}）— 成功率 {X}%，平均提升 {Y} 分
3. {策略名}（{维度}）— 成功率 {X}%，平均提升 {Y} 分
4. {策略名}（{维度}）— 成功率 {X}%，平均提升 {Y} 分
5. {策略名}（{维度}）— 成功率 {X}%，平均提升 {Y} 分
```

### 4. 团队能力图谱

按 **成员 x 维度** 统计平均提升分，并根据以下规则标注专业等级：
- 项目数 >= 10 且平均提升 >= 2.5 → **专家**
- 项目数 >= 5 且平均提升 >= 1.5 → **熟练**
- 其他 → **入门**

对每个维度，选出评级最高的成员作为「团队专家」。

输出格式：
```
🎯 团队能力图谱：

| 维度 | 团队专家 | 评级 | 平均提升 |
|------|---------|------|---------|
| {维度} | {成员名} | {专家/熟练/入门} | +{分数} |
| ... | ... | ... | ... |
```

### 5. 团队弱点检测

在所有项目中，找出 **平均最终分最低** 的维度，并进行深度分析。

输出格式：
```
⚠️ 团队弱点：

{维度名} 在所有项目中平均最终分 {X}（最低）
→ {Y}% 项目此维度未达标（低于及格线）
→ {有/无}成员评级为专家

建议：
1. 加强该领域知识学习
2. 考虑调整达标线或增加优化轮次
3. 可引入专攻此维度的自定义 Agent
```

### 6. 常见项目类型

统计共享仓库中所有项目的类型分布，按占比从高到低排列。

输出格式：
```
📁 最常见项目类型：{类型1} {百分比}% | {类型2} {百分比}% | {类型3} {百分比}% | ...
```

### 7. 清理临时目录

统计完成后，使用 Bash 删除临时克隆目录：
```bash
rm -rf .require-agent/sync-tmp/
```

输出 `"临时数据已清理。"` 确认清理完成。
