# 版本列表

你需要列出当前项目需求文档的所有版本信息和标签。

## 执行步骤

### 1. 定位最近项目

扫描 `.require-agent/projects/` 目录，找到最近修改的项目（按 state.json 的 `updated_at` 判断）。

如果没有项目：
  输出："未找到任何需求优化项目。请先使用 /require 启动一个项目。"
  终止。

### 2. 扫描版本文件

使用 Glob 扫描 `.require-agent/projects/{项目名}/versions/*.md`，获取所有版本文件列表。

如果 versions/ 目录为空或不存在：
  输出："该项目尚无版本记录。"
  终止。

### 3. 读取版本信息

读取 state.json 中的：
- `version_tags`：版本标签映射
- `round_history`：各轮次历史记录
- `current_round`：当前轮次

对每个版本文件，提取：
- 版本号（从文件名解析）
- 创建时间（从文件修改时间或 round_history 中获取）
- 对应轮次
- 标签（如有）

### 4. 展示版本列表

```
📋 项目「{项目名}」版本列表

| 版本 | 轮次 | 创建时间 | 标签 | 状态 |
|------|------|----------|------|------|
{每个版本一行，当前版本标记 ← 当前，有标签的显示标签名}

共 {N} 个版本，当前版本 v{当前版本号}
标签数：{version_tags 中的标签数量}
```
