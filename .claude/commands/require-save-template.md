# 保存为模板

你需要将当前项目的需求文档结构保存为可复用的模板。

## 执行步骤

### 1. 解析参数

从用户输入 `$ARGUMENTS` 中解析 `--as` 后的模板名。

如果参数缺失，提示用户：
```
用法：/require-save-template --as <模板名>
示例：/require-save-template --as 电商标准
```

### 2. 定位当前项目

扫描 `.require-agent/projects/` 目录，找到最近修改的项目。
读取项目的最终版本需求文档（`requirement-overview.md`）。

如果文档不存在，提示：
```
未找到需求文档。请确认项目已生成文档。
```

### 3. 生成模板

读取需求文档内容，进行模板化处理：
- 保留文档结构（标题层级、章节划分）
- 将业务特定内容替换为占位符说明（如 `[在此填写具体业务场景]`）
- 保留通用的章节结构、验收标准格式、非功能需求框架
- 在模板头部添加元信息：
  ```
  <!-- 模板信息 -->
  <!-- 来源项目：{项目名} -->
  <!-- 创建时间：{当前时间} -->
  <!-- 说明：基于 {项目名} 提炼的通用模板 -->
  ```

### 4. 保存模板

确保 `evolution/project-templates/` 目录存在（不存在则创建）。
将模板内容写入 `evolution/project-templates/{模板名}.md`。

如果同名模板已存在，提示用户确认是否覆盖。

### 5. 展示结果

```
💾 已保存为模板 {模板名}

来源项目：{项目名}
模板路径：evolution/project-templates/{模板名}.md
章节数量：{章节数}

使用方式：在 /require 启动新项目时可指定 --template {模板名}
```
