# 列出所有需求优化项目

你需要扫描所有需求优化项目，展示项目列表和状态概览。

## 执行步骤

### 1. 扫描项目目录

使用 Glob 扫描 `.require-agent/projects/*/state.json`，获取所有项目的状态文件路径。

如果 `.require-agent/projects/` 不存在或没有找到任何 `state.json`，提示：
```
未找到任何需求优化项目。

使用 /require "需求描述" 启动第一个项目。
```

### 2. 读取项目状态

逐个读取每个项目的 `state.json`，提取以下信息：
- `project`：项目名
- `phase`：当前阶段
- `current_round`：当前轮次
- `max_rounds`：最大轮次
- `scores`：各维度当前评分
- `target_score`：达标线
- `created_at`：创建时间
- `updated_at`：最近更新时间
- `enabled_agents`：已启用的 Agent 列表
- `stalled_dimensions`：停滞维度

### 3. 计算汇总指标

对每个项目计算：
- **平均分**：所有已启用维度评分的平均值（保留 1 位小数）
- **达标率**：已达标维度数 / 总启用维度数
- **阶段中文名**：将 phase 翻译为中文

阶段翻译映射：
| phase | 中文 | 图标 |
|-------|------|------|
| initialization | 初始化 | ... |
| breadth_scan | 广度扫描 | ... |
| deep_optimization | 深度优化 | ... |
| final_review | 终审 | ... |
| delivery | 交付中 | ... |
| completed | 已完成 | ... |

### 4. 展示项目列表

使用 Bash 工具获取每个项目目录的磁盘大小（`du -sh`）。

按 `updated_at` 倒序排列（最近更新的在前），展示：

```
需求优化项目列表

共 {总项目数} 个项目

| # | 项目名 | 阶段 | 轮次 | 平均分 | 达标率 | 创建时间 | 最近更新 | 大小 |
|---|--------|------|------|--------|--------|---------|---------|------|
| 1 | {项目名} | {阶段} | {轮次}/{最大轮次} | {平均分} | {达标率}% | {创建时间} | {更新时间} | {大小} |
| 2 | {项目名} | {阶段} | {轮次}/{最大轮次} | {平均分} | {达标率}% | {创建时间} | {更新时间} | {大小} |
...

汇总统计：
- 进行中：{phase 不是 completed 的项目数} 个
- 已完成：{phase 是 completed 的项目数} 个
- 最高平均分：{项目名}（{分数} 分）

可用操作：
  /require --resume {项目名}   ← 恢复某个项目
  /require-stats               ← 查看全局统计
  /require-clean               ← 清理项目
```

### 5. 空项目处理

如果某个项目的 `state.json` 读取失败或格式异常：
- 在列表中标注该项目为"状态异常"
- 不影响其他项目的展示
