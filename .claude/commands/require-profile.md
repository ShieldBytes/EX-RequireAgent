# 用户画像

你需要展示当前用户的使用画像统计，或根据需要重置画像。

## 执行步骤

### 1. 解析参数

从用户输入 `$ARGUMENTS` 中检查是否有 `--reset` 参数。

### 2. 如果有 --reset

删除或清空 `evolution/user-profile.json`，展示：
```
🔄 用户画像已清除。将在后续使用中重新积累。
```
终止。

### 3. 收集统计数据

扫描 `evolution/projects/` 和 `.require-agent/projects/` 目录，统计：
- **项目总数**：所有项目数量
- **已完成项目数**：phase 为 `completed` 或 `archived` 的项目
- **进行中项目数**：其他状态的项目
- **平均满意度**：从各项目最终评分中计算平均分
- **擅长维度**：统计每个维度的平均得分，找出得分最高的 3 个维度
- **薄弱维度**：找出得分最低的 2 个维度
- **平均优化轮次**：已完成项目的平均 current_round
- **常用模板**：如有使用模板的记录

读取 `evolution/user-profile.json`（如存在），合并历史数据。

### 4. 展示用户画像

```
👤 用户画像

📊 项目统计
  总项目数：{N}
  已完成：{N}  进行中：{N}  已归档：{N}

📈 质量趋势
  平均最终评分：{分数}
  平均优化轮次：{轮次}

💪 擅长维度（平均分最高）
{前3个维度及平均分}

⚠️ 待提升维度（平均分最低）
{后2个维度及平均分}

🏷️ 常用标签
{统计最常用的版本标签类型}

📅 活跃度
  首次使用：{最早项目时间}
  最近使用：{最近项目时间}
```
