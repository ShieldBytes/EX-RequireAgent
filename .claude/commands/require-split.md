# 文档模块化拆分

你需要将单一的需求文档拆分为按功能模块组织的独立文件。

## 执行步骤

### 1. 定位当前项目

扫描 `.require-agent/projects/` 目录，找到最近修改的项目。
读取 state.json。

如果 state.json 中 `modular` 已经为 `true`：
```
该项目文档已完成模块化拆分。模块目录：modules/
如需重新拆分，请先手动将 state.json 中 modular 设为 false。
```

### 2. 读取主文档

读取项目的 `requirement-overview.md`，分析文档结构：
- 识别一级和二级标题作为模块边界
- 识别功能模块章节（如"用户管理"、"订单系统"等）
- 统计各章节的内容量

### 3. 拆分文档

在项目目录下创建 `modules/` 子目录。

按功能模块章节拆分：
- 每个功能模块生成独立文件：`modules/{模块名}.md`
- 文件内容包含该模块的完整需求描述
- 每个模块文件头部添加引用信息：
  ```
  <!-- 模块：{模块名} -->
  <!-- 拆分自：requirement-overview.md -->
  <!-- 拆分时间：{当前时间} -->
  ```

### 4. 重构主文档

将原 `requirement-overview.md` 改写为概览格式：
- 保留项目概述、背景、整体架构等通用部分
- 各功能模块章节替换为模块引用链接：
  ```
  ## {模块名}
  > 详见 [modules/{模块名}.md](modules/{模块名}.md)
  >
  > 概要：{模块的一句话描述}
  ```

### 5. 更新状态

更新 state.json：
- 设置 `modular` 为 `true`
- 添加 `modules` 数组，列出所有模块名
- 更新 `updated_at`

### 6. 展示结果

```
📂 文档模块化拆分完成

项目：{项目名}
拆分模块数：{N}

模块列表：
{每个模块一行：模块名 — 内容大小}

主文档已改为概览格式，各模块独立维护。
```
