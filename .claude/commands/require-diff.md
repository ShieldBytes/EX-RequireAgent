# 版本差异对比

你需要对比需求文档的两个版本，生成清晰的差异摘要。

## 执行步骤

### 1. 解析参数

从用户输入 `$ARGUMENTS` 中解析两个版本号，格式为 `v{数字}`，如 `v3 v7`。

如果参数缺失或格式不正确，提示用户：
```
用法：/require-diff <版本1> <版本2>
示例：/require-diff v3 v7
```

将较小版本号记为"旧版本"，较大版本号记为"新版本"。

### 2. 定位最近项目

使用 Glob 扫描 `.require-agent/projects/*/state.json`，找到最近修改的项目。

如果没有找到任何项目，提示：
```
未找到任何需求优化项目。请先使用 /require 启动一个项目。
```

### 3. 读取版本文件

读取以下两个文件：
- `.require-agent/projects/{项目名}/versions/{旧版本}.md`
- `.require-agent/projects/{项目名}/versions/{新版本}.md`

如果任一文件不存在，提示：
```
版本 {版本号} 不存在。
可用版本：{使用 Glob 列出 versions/*.md 下所有文件}
```

### 4. 分析差异

逐章节对比两个版本的内容，生成差异摘要。分析以下方面：

1. **新增的需求项**：新版本中有但旧版本中没有的需求
2. **修改的需求项**：两个版本中都有但内容发生变化的需求
3. **删除的需求项**：旧版本中有但新版本中没有的需求
4. **结构变化**：章节增减、模板升级等
5. **关键词变化**：新增或移除的关键术语

### 5. 展示差异报告

```
版本对比：{旧版本} vs {新版本}
项目：{项目名}

---

## 新增（{数量}项）

- [新增] {需求项描述}
- [新增] {需求项描述}
...

## 修改（{数量}项）

- [修改] {需求项名称}
  旧：{旧内容摘要}
  新：{新内容摘要}
...

## 删除（{数量}项）

- [删除] {需求项描述}
...

## 结构变化

- {结构变化描述，如"从大纲模板升级为 PRD 模板"}
...

---

总计：+{新增数} ~{修改数} -{删除数}
```

如果两个版本完全相同，展示：
```
版本 {旧版本} 和 {新版本} 内容完全一致，无差异。
```
