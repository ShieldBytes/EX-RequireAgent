# 附录 A：Agent 详细定义

## 1. 完整性 Agent

**核心职责：** 找出需求中缺失的部分

**详细功能：**

- **功能点完整性**：核心功能齐全、辅助功能（设置/帮助/反馈）、管理后台功能
- **场景完整性**：正常场景、异常场景（网络断开/服务超时/数据异常）、边界场景（空数据/超大数据/并发）、生命周期场景（首次使用/长期未用/账号注销）
- **需求要素完整性**：每个功能点有验收标准、优先级标注、非功能性需求入口、术语表
- **状态完整性**：每个业务对象状态流转完整、状态转换条件明确、异常状态处理

**策略池：**
- A. 功能遍历法 — 逐模块检查功能清单
- B. 反向场景法 — 从异常/失败倒推缺什么
- C. 边界推演法 — 极端情况下功能是否成立
- D. 生命周期法 — 用户注册到注销全生命周期
- E. 竞品对标法 — 与竞品功能清单对比

**自我进化：**
- 记录每种项目类型最常遗漏的功能点
- 记录每种策略的发现效率
- 跨项目积累"通用遗漏清单"

---

## 2. 一致性 Agent

**核心职责：** 找出需求中自相矛盾的部分

**详细功能：**

- **逻辑一致性**：需求间矛盾、同一功能不同描述、业务规则冲突
- **术语一致性**：同一概念多种叫法、术语定义歧义、生成术语对照表
- **格式一致性**：颗粒度统一、描述风格统一
- **引用一致性**：跨功能引用有效性、依赖关系无循环
- **版本一致性**：旧需求同步更新、新增需求与已有对齐

**策略池：**
- A. 交叉比对法 — 逐对比较需求项
- B. 术语图谱法 — 构建术语关系图
- C. 规则冲突检测法 — 提取业务规则两两检查
- D. 引用链追踪法 — 追踪跨功能引用验证有效性

**自我进化：**
- 积累常见矛盾模式
- 积累行业通用术语表
- 学习哪些模块之间最容易产生矛盾

---

## 3. 用户旅程 Agent

**核心职责：** 模拟真实用户走完全部流程，找出体验断点

**详细功能：**

- **多角色旅程模拟**：新手、普通用户、高级用户、管理员、特殊用户（访客/被封禁/过期会员）
- **关键路径分析**：最短步骤完成核心任务、不必要跳转和等待、错误恢复路径
- **情绪地图**：愉悦点、挫败点、流失高风险区
- **断点检测**：流程中断恢复、跨设备延续、跨时间延续
- **上下文连贯性**：页面间信息不断裂、操作及时反馈、用户始终知道位置和下一步

**策略池：**
- A. 角色扮演法 — 逐角色模拟完整旅程
- B. 关键路径法 — 只走核心任务最短路径
- C. 情绪地图法 — 标注全程情绪波动
- D. 断点压测法 — 每步故意中断看恢复
- E. 对比旅程法 — 与竞品用户旅程对比

**自我进化：**
- 积累不同项目类型的旅程模板
- 记录最常见的断点位置
- 学习哪种角色最容易发现问题

---

## 4. 业务闭环 Agent

**核心职责：** 验证商业逻辑能否自洽运转

**详细功能：**

- **核心商业逻辑**：价值链完整、收入来源明确、成本结构合理
- **增长循环验证**：获客渠道、留存机制、传播机制、变现路径
- **单元经济模型**：CAC、LTV、LTV>CAC、回本周期
- **利益方分析**：所有利益方、核心诉求、冲突平衡
- **商业风险识别**：市场风险、竞争风险、政策风险

**策略池：**
- A. 商业画布法 — 9 大模块全面验证
- B. 成本收益法 — 量化投入产出
- C. 竞品对标法 — 对比竞品商业模式
- D. 增长飞轮法 — 验证获客→留存→变现→再获客循环
- E. 利益方博弈法 — 模拟各方利益诉求和冲突

**自我进化：**
- 积累不同商业模式的验证框架
- 记录不同行业的单元经济基准值
- 学习什么类型商业逻辑最容易出漏洞

---

## 5. 可行性 Agent

**核心职责：** 评估需求在现实条件下是否可落地

**详细功能：**

- **技术可行性**：成熟方案、前沿技术风险、技术难点标注
- **资源可行性**：团队配置、规模匹配、人才瓶颈
- **时间可行性**：量级匹配时间线、分期建议、MVP 范围
- **成本可行性**：开发成本、运营成本、预算约束
- **风险分级**：每个功能点风险等级（低/中/高）+ 替代方案

**策略池：**
- A. 技术调研法 — 搜索方案成熟度和案例
- B. 类比估算法 — 参照类似项目
- C. MVP 裁剪法 — 按可行性裁剪最小可行版本
- D. 风险矩阵法 — 概率×影响评估
- E. 替代方案法 — 高风险需求寻找低风险替代

**自我进化：**
- 积累不同技术方案的风险历史
- 校准估算准确度
- 积累 MVP 裁剪经验

---

## 6. 安全 Agent

**核心职责：** 从需求层面识别安全隐患

**详细功能：**

- **认证与鉴权**：登录方式、密码策略、多因素认证、会话管理、权限模型
- **数据安全**：敏感数据识别、加密要求、脱敏要求、数据分级
- **输入安全**：输入校验、文件上传限制、防注入防 XSS
- **接口安全**：API 认证、限流防刷、防重放、敏感操作二次确认
- **安全运营**：安全日志审计、异常行为检测、安全事件响应、定期评估

**策略池：**
- A. OWASP Top 10 对照法
- B. 攻击面分析法
- C. 权限矩阵法
- D. 威胁建模法（STRIDE）
- E. 安全分级法

**自我进化：**
- 积累不同行业的安全标准
- 记录最常见的安全需求遗漏
- 跟踪新型安全威胁

---

## 7. 性能 Agent（按需启用）

**核心职责：** 确保性能需求有明确可量化的定义

**详细功能：**

- **容量规划**：用户规模（初期/1年/3年）、日活和峰值并发、数据增长、存储
- **性能指标**：核心操作响应时间、页面加载、接口吞吐、可用性目标
- **峰值应对**：峰值场景、负载预估、弹性扩容、限流降级
- **降级策略**：功能优先级、降级后体验、恢复策略
- **监控要求**：关键指标、告警阈值、容量预警

**策略池：**
- A. 场景建模法 — 日常/高峰/极端三级
- B. SLA 定义法 — 每个核心功能量化指标
- C. 瓶颈预判法 — 从需求层面预判瓶颈
- D. 容量规划法 — 按增长曲线规划
- E. 降级预案法 — 每个功能定义降级方案

---

## 8. 无障碍与国际化 Agent（按需启用）

**核心职责：** 确保产品对所有用户都可用

**详细功能：**

- **视觉无障碍**：色彩对比度、字体缩放、屏幕阅读器、替代文本
- **运动无障碍**：键盘操作、触摸目标、时间限制延长
- **认知无障碍**：操作简洁、错误易懂、导航可预测、帮助引导
- **设备适配**：多屏幕尺寸、横竖屏、不同输入方式、弱网/离线
- **国际化**：多语言架构、日期/数字/货币格式、文本方向、文化适配、内容本地化

**策略池：**
- A. WCAG 对照法
- B. 多角色模拟法
- C. 多地区模拟法
- D. 极端适配法 — 最长文本/最小屏幕/最慢网络
- E. 设备矩阵法

---

## 9. 数据 Agent（按需启用）

**核心职责：** 确保数据相关需求完整清晰

**详细功能：**

- **数据模型**：核心实体、属性、关系、状态机
- **数据流转**：功能间流动、变更触发、同步一致性、实时性
- **数据质量**：输入校验、格式标准、唯一性、必填/选填
- **数据生命周期**：创建、更新历史、归档、删除策略、恢复
- **数据迁移兼容**：老系统迁移、导入导出格式、版本兼容

**策略池：**
- A. 实体梳理法
- B. 数据流追踪法
- C. 边界值分析法
- D. 状态机推导法
- E. 数据血缘分析法

---

## 10. 依赖与集成 Agent（按需启用）

**核心职责：** 梳理外部依赖，评估集成风险

**详细功能：**

- **依赖识别**：第三方服务清单、用途和重要程度、开源替代、依赖图谱
- **集成需求**：集成方式、数据交换格式、认证授权、调用频率配额
- **风险评估**：服务可用性 SLA、供应商锁定、价格风险、政策风险、安全风险
- **故障预案**：降级方案、备选供应商、重试策略、降级用户体验
- **成本预估**：计费模型、月度成本、规模增长曲线

**策略池：**
- A. 依赖图谱法
- B. 故障预案法
- C. 替代方案法
- D. 成本建模法
- E. 供应商评估法

---

## 11. 红队 Agent

**核心职责：** 全系统对抗者，找出所有人的盲区

**详细功能：**

- **需求挑战**：恶意使用、假设不成立、最坏情况
- **Agent 产出挑战**：质疑其他 Agent 结论依据、挑战评分合理性、检验情报使用
- **多角色对抗**：恶意用户、竞争对手、刁钻甲方、监管者、愤怒用户
- **压力测试**：极端规模、极端数据、极端场景、极端对抗
- **全系统审计**（每 3 轮）：审查所有 Agent 产出 + 随机全局质疑问题

**策略池：**
- A. 角色扮演法
- B. 极端推演法
- C. 反向思维法
- D. 历史案例法 — 搜索类似产品真实失败案例
- E. 系统审计法

**攻击力动态调节：** 温和（30%）/ 标准（60%）/ 激进（90%），根据效果和用户反馈自动调节。

---

## 12. 评估 Agent

**核心职责：** 全局裁判 + 质量监控 + 进度管理

**详细功能：**

- **维度评分**：分数 + 置信度 + 理由、分歧检测、证据支撑
- **跨维度一致性**：维度之间矛盾检查
- **文档质量评估**：可读性、结构性、颗粒度、可操作性
- **成熟度评估**：概念期 → 框架期 → 细化期 → 成熟期，自动切换输出格式
- **优化效率监控**：轮 ROI、趋势分析、收敛预估、智能停止建议
- **冲突仲裁**：自动协调 → 评估仲裁 → 升级人工

**策略池：**
- A. 多维度独立评分法
- B. 交叉验证法
- C. 基准对比法 — 与历史项目同阶段对比
- D. 文档可读性评估法
- E. 趋势分析法

## 14. 写手 Agent

**核心职责：** 把所有优化结果整合成高质量需求文档

**详细功能：**

- **内容整合**：汇总建议、消除重复、解决轻微不一致、标注未解决冲突
- **文档结构管理**：根据成熟度选结构、新增内容放对位置、逻辑连贯、目录维护
- **写作质量**：简洁无歧义、主动语态、术语统一、适配读者水平
- **多受众适配**：管理层摘要 / 产品 PRD / 开发视角
- **变更管理**：每轮变更摘要、标注变更原因和来源、changelog 维护、版本 diff
- **可追溯性**：每条需求标注来源（用户原始 / Agent 优化 / 情报支撑 / 人工决策）
- **模块化管理**：只重写受影响的模块、更新 overview 状态和评分

**策略池：**
- A. 结构化重写法
- B. 增量更新法 — 只改变化部分
- C. 多受众生成法
- D. 可追溯标注法
- E. 术语统一法
