# 模块化文档管理

本模块定义模块化检测、拆分执行、模块化模式下的评分调整和优化调整。

---

## 一、模块化检测（步骤 2.7.1）

在广度扫描建立范围基线后（步骤 2.7 之后）执行。

统计 v2 文档中的功能模块数量（按一级功能章节计数）：

- 模块数 <= 3 -> 保持单文件，跳过。在 state.json 中标记 `"modular": false`
- 模块数 4-8 -> 向用户建议：
  "检测到 {N} 个功能模块，建议拆分为模块化文档。拆分？(Y/N)"
  Y -> 执行拆分  N -> 保持单文件
- 模块数 > 8 -> 自动拆分

### 执行拆分

1. 创建 {output_dir}/modules/ 目录
2. 将每个功能模块章节提取为独立文件：modules/01-{模块名}.md
3. 将 requirement-overview.md 改写为概览（用 templates/requirement-overview-modular.md 模板）
4. 在 state.json 中标记 `"modular": true`

## 二、模块化模式下的评分调整（两层评分）

当 modular = true 时，评分采用两层结构：

### 模块级评分

对每个模块文件独立评分：

```json
{
  "module": "{模块名}",
  "scores": {
    "completeness": <分数>,
    "consistency": <分数>,
    "feasibility": <分数>,
    "user_journey": <分数>,
    "business_loop": <分数>
  },
  "feature_count": <该模块功能点数>
}
```

评分时向 evaluator 发送单个模块文件内容（而非整个文档）。

### 项目级评分（加权汇总）

项目整体评分 = 各模块评分的加权平均值

权重 = 模块功能点数 / 总功能点数

```
项目维度分 = SUM(模块维度分 * 模块权重)
```

在 scores/round-N.json 中同时记录模块级和项目级评分：

```json
{
  "round": {N},
  "project_scores": { ... },
  "module_scores": [
    { "module": "01-xxx", "scores": { ... }, "weight": 0.3 },
    { "module": "02-yyy", "scores": { ... }, "weight": 0.7 }
  ]
}
```

## 三、模块化模式下的优化调整（两级目标选择）

步骤 3.1 选择目标维度时，模块化模式使用两级选择：

### 第一级：选择目标模块

1. 计算每个模块的最低分维度分数
2. 选出最低分模块（功能点数加权）
3. 如果多个模块最低分相同，选功能点数最多的（优先优化核心模块）

### 第二级：选择目标维度

在目标模块内，按原有的并列处理规则选择最低分维度。

### 优化范围限定

后续步骤 3.2-3.7 只读取和修改目标模块文件（modules/XX-{模块名}.md），不修改其他模块。

向用户展示：
```
深度优化 — 第 {N}/{max} 轮
目标模块：{模块名}（{功能点数} 个功能点）
目标维度：{维度名}（当前 {分数} 分）
```

## 四、模块生命周期

模块化文档的完整生命周期：

| 阶段 | 操作 |
|------|------|
| 创建 | 步骤 2.7.1 拆分时创建 |
| 评分 | 每轮对目标模块单独评分 |
| 优化 | 每轮只优化一个模块的一个维度 |
| 一致性检查 | 每 3 轮执行跨模块一致性检查（步骤 3.7.1） |
| 合并概览 | 终审时更新 requirement-overview.md 概览，确保与各模块一致 |
| 交付 | 最终交付 overview + modules/ 目录 |

### 跨模块一致性检查（步骤 3.7.1）

每 3 轮触发一次（current_round % 3 == 0）：

1. 读取概览文档 requirement-overview.md
2. 读取各模块文件的前 20 行（摘要）
3. 调度 consistency Agent 检查：
   - 模块间术语是否统一
   - 业务规则是否矛盾
   - 依赖关系是否准确
4. 如发现问题 -> 在下一轮优先处理该一致性问题

### 模块新增/删除

- 如果优化过程中发现需要新增模块：创建新文件，更新概览
- 如果发现某模块应合并：合并文件，更新概览
- 任何模块结构变更后，重新计算权重
